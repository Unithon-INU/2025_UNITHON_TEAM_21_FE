# Commit Convention

커밋 메시지를 작성하실 때 자세히 작성해준시다면 효율적인 작업에 도움이 됩니다.

```
커밋 컨벤션 예시
git commit -m "[Feat] 카카오 api 추가"
git commit -m "[Docs] /src/asset/charater.svg 추가"
```

- `Feat` : 새로운 기능 추가
- `Fix` : 버그 수정
- `Docs` : 문서 수정
- `Style` : 코드 포맷팅, 세미콜론 누락, 코드 변경이 없는 경우
- `Refactor` : 코드 리펙토링할 시
- `Etc` : 기타
